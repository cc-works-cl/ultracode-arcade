# ◆ ULTRA ARCADE

ゲーム・便利ツール・音楽スタジオを **1ページ** に集めた、レスポンシブ（PC / スマホ / タブレット、タッチ操作対応）な静的 Web アプリ。
HTML / CSS / vanilla JS のみ、**ビルド不要**。任天堂 Switch のホーム画面風の遊び心（角丸タイル・明るい原色・白基調・ぷにっとした手触り・効果音）で統一したオリジナルデザイン。

### 🎹 スタジオ（多重録音ルーパー）
16ステップ × 4レーン（キック / スネア / ハイハット / クラップ）のドラムマシン。Web Audio API でドラムを合成し、ルックアヘッド方式で正確にスケジューリング。リズムテンプレ（8ビート / 4つ打ち / ヒップホップ / ファンク / シャッフル等）を切替・編集。`getUserMedia` + `MediaRecorder` で声などをループに重ね録りし、複数トラックを重ねられる。

## 動かし方

### そのまま開く
ほとんどの機能は `index.html` をブラウザで直接開くだけで動きます。

### ローカルサーバ（推奨）
スタジオの**録音機能**は `file://` では動きません（後述）。ローカルで全機能を試すには簡易サーバ経由で：

```sh
cd ultracode-arcade
python3 -m http.server 8765
# → http://127.0.0.1:8765/ を開く
```

## GitHub Pages へ配信

1. このリポジトリを GitHub に push。
2. リポジトリの **Settings → Pages** で *Source* を `Deploy from a branch`、ブランチを `main`（フォルダ `/root`）に設定。
3. 数分後 `https://<ユーザー名>.github.io/<リポジトリ名>/` で公開されます。

- ルートに `.nojekyll` を置いてあるので Jekyll の処理は走りません。
- 全アセットは相対パス。サブパス配信でも動作します。

## ⚠️ 録音機能について
スタジオの重ね録り（`getUserMedia` / `MediaRecorder`）は、ブラウザの仕様上 **HTTPS か localhost でのみ** 利用できます。
- ✅ GitHub Pages（https）／`localhost`：録音OK
- ❌ `file://` で直接開いた場合：録音UIは自動で無効化し、案内を表示します（ドラム・トラック再生など他機能は動作）。

また `navigator.vibrate`（触覚フィードバック）は Android のみ対応で、iOS Safari では無反応です。本アプリの「ぷにっとした手触り」は視覚的な押下アニメ＋効果音を主役にしているため、振動非対応端末でも体験は損なわれません。

## アーキテクチャ（並列開発できる独立モジュール構成）

- `index.html` … エントリ。`core.js` を最初に読み、各コンポーネントの `<script>` を列挙。
- `css/base.css` … デザインシステム（トークン・タイル・タブ・ぷにっとアニメ）。
- `js/core.js` … ランタイム。タブ/タイルUI・画面遷移・効果音エンジン（Web Audio合成）・`ctx` ヘルパー・登録レジストリ。
- 各コンポーネントは **1ファイル = 1モジュール**。`Arcade.register({...})` で自己登録するだけ。互いに依存しないため**並列開発可能**。
- 開発コントラクトは [`CONTRACT.md`](./CONTRACT.md)、正典テンプレートは `js/tools/counter.js`。

```
index.html
.nojekyll
css/base.css
js/
  core.js
  games/   breakout.js  snake.js  arrowsgo.js  sokoban.js  thread.js
  tools/   json.js  counter.js  color.js  markdown.js  password.js
  studio/  studio.js
```
